# RepleyPass 需求整理

## 目标

基于参考面板，重新实现一个全新的中转面板。当前第一阶段先完成需求整理、技术栈调研、前端/API 边界设计，不进入完整业务实现。

## 产品定位

RepleyPass 面向中转服务的运维管理，核心是把入口设备组、出口设备组、转发规则、用户/设备、在线连接、真实 IP、证书和运行事件统一到一个后台中。

## MVP 范围

第一版先实现前端与 API，不要求真实设备上线。

### 管理端基础

- 管理员登录、登出、会话保持。
- 当前登录用户信息。
- 全局搜索。
- 页面级刷新。
- 统一空状态、加载状态、错误状态。

### 仪表盘

- 总览指标：
  - 在线节点数。
  - 活跃连接数。
  - 今日流量。
  - 真实 IP 捕获率。
  - 告警数量。
  - 规则同步失败数量。
- 实时/准实时拓扑：
  - 入口设备组。
  - 策略组。
  - 出口设备组。
- 最近事件：
  - 节点上下线。
  - 健康检查异常。
  - 规则变更。
  - 证书续期。

### 设备/节点管理

- 设备列表。
- 设备详情。
- 设备状态：
  - 在线。
  - 离线。
  - 告警。
  - 禁用。
- 设备指标：
  - 负载。
  - 延迟。
  - 连接数。
  - 入站/出站流量。
  - 最后心跳时间。
- 设备组：
  - 入口设备组。
  - 出口设备组。
  - 故障转移组。
  - 用户组可用范围。
- 添加、编辑、禁用、删除设备。
- 高级 JSON 配置编辑。

### 中转规则

- 规则列表。
- 新建、编辑、暂停、启动、删除规则。
- 入口配置：
  - 入口设备组。
  - 监听地址。
  - 监听端口或端口段。
  - 随机端口。
  - 协议。
- 出口配置：
  - 单出口。
  - 出口设备组。
  - 限制出口。
  - 权重。
- 策略：
  - Round Robin。
  - IP Hash。
  - Least Conn。
  - Fallback。
- 同步状态：
  - 未同步。
  - 正常。
  - 同步失败。
  - 已暂停。
- 真实 IP：
  - Proxy Protocol 关闭。
  - Proxy Protocol v1。
  - Proxy Protocol v2。
  - 接收/发送模式。
- 运行统计：
  - 当前连接数。
  - 今日流量。
  - 错误数。
  - 最近命中时间。
- 批量规则：
  - 导入。
  - 导出。
  - JSON 格式。

### 用户管理

- 用户列表。
- 用户详情。
- 用户状态：
  - 正常。
  - 过期。
  - 流量不足。
  - 禁用。
- 用户组。
- 用户流量：
  - 总流量。
  - 已用流量。
  - 今日流量。
- 转发规则归属用户。
- 流量倍率：
  - 入口倍率。
  - 出口倍率。

### 设备组管理

- 入口组：
  - 共用入口规则。
  - 入站屏蔽配置。
  - TLS 入站策略。
  - UDP 设置。
  - 反向隧道设置。
- 出口组：
  - 连接方式。
  - 连接地址/端口。
  - 最小连接数负载均衡。
  - 健康检查。
  - 故障转移组。
- 高级配置 JSON 编辑。

### 在线 IP

- 在线来源 IP 列表。
- 按规则、入口节点、时间范围过滤。
- 连接数排行。
- 最近活跃时间。
- 真实 IP 来源标识：
  - Proxy Protocol。
  - 连接日志。
  - 回退识别。

### 证书管理

- 证书列表。
- 证书详情。
- 域名、签发方、到期时间、剩余天数。
- 自动续期状态。
- 使用该证书的规则。
- 新增/更新证书。

### 探针/健康检查

- 探针列表。
- 探针结果。
- 健康检查失败原因。
- TCP 握手超时/refused 统计。
- 故障转移触发记录。

### 日志与审计

- 操作日志：
  - 登录。
  - 新建/修改/删除规则。
  - 设备/设备组变更。
  - 证书变更。
- 事件日志：
  - 设备告警。
  - 健康检查。
  - 规则运行异常。
- 支持关键词、类型、时间范围过滤。

## 非 MVP

- 远程命令执行。
- 真实设备接入上线。
- 复杂租户计费。
- 公开用户自助购买页面。
- 大规模报表导出。

## 角色与权限

第一版先保留两类角色：

- 超级管理员：全部权限。
- 只读管理员：可查看数据，不可修改配置。

后续可扩展：

- 运维管理员。
- 审计员。
- 租户管理员。

## 数据生命周期

参考站每日 0 点（UTC+8）清空演示数据。RepleyPass 实现时建议拆分：

- 长期保留：
  - 管理员账号。
  - 设备配置。
  - 设备组配置。
  - 规则配置。
  - 证书配置。
  - 系统设置。
- 可按策略清理：
  - 在线连接快照。
  - 实时指标。
  - 事件日志。
  - 访问日志。
  - 流量统计明细。

## 前端建议

第一阶段建议做成单页管理后台：

- 左侧导航。
- 顶部工具栏。
- 主工作区。
- 表格、抽屉、弹窗、详情页组合。
- 指标和拓扑用轻量图表/自绘组件。

视觉方向：

- 运维工具风格，信息密度高，减少营销化装饰。
- 状态色清晰：
  - 绿色：正常。
  - 黄色：告警。
  - 红色：故障。
  - 灰色：禁用/离线。
- 表格操作要支持批量处理。

## 后端/API 建议

第一阶段先提供 mock/内存数据 API，保证前端工作流完整。

建议统一响应：

```json
{
  "success": true,
  "data": {},
  "message": "",
  "requestId": "req_xxx"
}
```

分页响应：

```json
{
  "success": true,
  "data": {
    "items": [],
    "total": 0,
    "page": 1,
    "pageSize": 20
  },
  "message": "",
  "requestId": "req_xxx"
}
```
